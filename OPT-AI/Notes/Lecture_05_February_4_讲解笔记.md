### 第 ① 页：Single Dimensional Optimization（续）

**原文翻译：**

单维优化（续）。两幅图展示区间更新规则：当 f(λᵏ) ≤ f(ρᵏ) 时，新区间为 [aᵏ, ρᵏ]；当 f(ρᵏ) < f(λᵏ) 时，新区间为 [λᵏ, bᵏ]。

**讲解：**

本页是 Lecture 4 最后两页的直接延续，教授在新一讲的开头重新画出区间缩减的两种情况，帮助学生"热身"回忆。这种教学策略在每次课的开头都会出现——用前一讲的最后内容作为本讲的出发点，确保知识的连贯性。

两种情况的逻辑核心是**单峰函数的排除法**：如果左试探点的函数值更小（f(λᵏ) ≤ f(ρᵏ)），那么最小值不可能在 ρᵏ 右边（因为函数在那里已经开始上升），因此丢弃右段。反之亦然。这个思想与二分查找在排序数组中搜索元素的思路完全类似——每次比较都排除一半的可能性。

关键观察：无论哪种情况发生，区间都会缩小。但缩小多少取决于 λᵏ 和 ρᵏ 的位置——这正是不同算法（二分搜索、黄金分割等）的差异所在。

### 第 ② 页：Bisection Search for Single-Dimensional Optimization

**原文翻译：**

单维最小化的二分搜索。配图展示 λᵏ 和 ρᵏ 对称放置在中点两侧、间距为 ε 的布局。算法四步：①选初始区间和停止容差；②计算 λᵏ = (aᵏ+bᵏ)/2 − ε、ρᵏ = (aᵏ+bᵏ)/2 + ε；③按比较结果更新区间；④检查收敛或继续。

**讲解：**

这一页给出了二分搜索的完整算法描述，是上一讲"概念引入"到"可执行算法"的转化。算法结构非常清晰：初始化→计算试探点→比较并更新→检查终止。

核心参数有两个：ε（探测间距）和 θ（停止容差）。ε 控制两个试探点的距离——太小会导致数值比较不稳定（浮点精度问题），太大则每步缩减不够充分。θ 决定何时停止迭代——当区间宽度小于 θ 时，我们认为已经足够精确地找到了最小值。

每次迭代后，区间宽度从 (bᵏ − aᵏ) 缩减为大约 (bᵏ − aᵏ)/2 + ε。当 ε 很小时，近似为二分（每步减半）。经过 N 次迭代，区间宽度约为初始宽度的 (1/2)ᴺ——这是指数级收敛，与二分查找的 O(log N) 复杂度一致。

但每次迭代需要**两次函数求值**——这引出了下一页的效率问题。

### 第 ③ 页：二分搜索示例与动机

**原文翻译：**

使用二分搜索最小化 x² + 2x，搜索区间为 [−3, 5]。二分搜索每次迭代需要两次函数求值：f(λᵏ) 和 f(ρᵏ)。问：能否每次迭代只做一次函数计算？

**讲解：**

教授用一个简单的二次函数 f(x) = x² + 2x 作为练习例子。这个函数的最小值在 x = −1 处（令 f'(x) = 2x + 2 = 0），最小值为 f(−1) = −1。学生可以手动执行几步二分搜索来验证算法确实逐步逼近 x = −1。

但本页真正的重点是最后一句话："Can we get away with one function computation at each iteration?"这个问题看似简单，实则深刻。在二分搜索中，每一步的两个试探点 λᵏ 和 ρᵏ 都是全新计算的，上一步的函数值无法复用。如果我们能巧妙安排试探点的位置，使得本次的某个试探点恰好是上次的某个试探点，就能节省一次函数求值。

这个"复用"的想法直接导向了黄金分割搜索——数学上最优雅的单维优化算法之一。教授用这个问题吊起胃口，为下一页的黄金分割搜索做铺垫。

### 第 ④ 页：Golden Section Search for One-Dimensional Optimization

**原文翻译：**

黄金分割搜索。能否安排左右试探点，使得 λᵏ⁺¹ 匹配 ρᵏ（情况★），或 ρᵏ⁺¹ 匹配 λᵏ（情况★★）？两幅图分别展示这两种情况：在情况★中，更新后只需计算 f(ρᵏ⁺¹)；在情况★★中，只需计算 f(λᵏ⁺¹)。此外，要求区间每步缩减相同比例 β。

**讲解：**

这是黄金分割搜索的核心设计页面，教授提出了两个同时满足的条件：

**条件一（试探点复用）：** 当区间更新后，旧的某个试探点恰好变成新的某个试探点。具体来说：如果情况★发生（f(λᵏ) ≤ f(ρᵏ)，区间变为 [aᵏ, ρᵏ]），那么新区间的左试探点 λᵏ⁺¹ 应该等于旧的 ρᵏ——这样只需计算一个新的 ρᵏ⁺¹。反之亦然。这正是回答第③页问题的关键——每步只需**一次**新函数求值。

**条件二（等比缩减）：** 区间每步缩减同一比例 β，即 bᵏ⁺¹ − aᵏ⁺¹ = β·(bᵏ − aᵏ)。这保证了算法行为的一致性和可预测性——每步的"效率"相同。

两幅图非常清楚地展示了这种"复用"机制：旧的试探点在新区间中保持了其"相对位置"。这种对称性暗示了黄金比的出现——数学中最优美的常数之一即将登场。

---

### 第 ⑤ 页：黄金分割比推导（情况 ★）

**原文翻译：**

从情况★和★★出发推导 λᵏ 和 ρᵏ 的表达式。在情况★下，等比缩减条件给出 λᵏ = β·aᵏ + (1−β)·bᵏ。在情况★★下，给出 ρᵏ = (1−β)·aᵏ + β·bᵏ。匹配条件（情况★中 λᵏ⁺¹ = ρᵏ）导出方程 1 − β = β²，即 β² + β − 1 = 0，解得 β = (−1+√5)/2。

**讲解：**

这一页是纯粹的代数推导，但背后的逻辑非常自然。

**第一步：从等比缩减条件推导试探点位置。** 以情况★为例：新区间 [aᵏ, ρᵏ] 的宽度为 ρᵏ − aᵏ = β(bᵏ − aᵏ)，因此 ρᵏ = aᵏ + β(bᵏ − aᵏ) = (1−β)aᵏ + βbᵏ。类似地，情况★★给出 λᵏ = βaᵏ + (1−β)bᵏ。注意 λᵏ 和 ρᵏ 的表达式是对称的——只是 β 和 1−β 互换了位置。这种对称性反映了两个试探点关于区间中点对称。

**第二步：从匹配条件推导 β 的值。** 在情况★中，要求 ρᵏ = λᵏ⁺¹。将 λᵏ⁺¹ 用 aᵏ⁺¹ = aᵏ 和 bᵏ⁺¹ = ρᵏ 展开，然后代入 λᵏ 和 ρᵏ 的表达式，最终化简得到 (1−β)(aᵏ−bᵏ) = β²(aᵏ−bᵏ)。由于 aᵏ ≠ bᵏ，可以消去得到 1 − β = β²，即 **β² + β − 1 = 0**。

这是一个一元二次方程，正根为 β = (−1+√5)/2 ≈ 0.618——正是**黄金比**！这不是巧合，而是"等比缩减 + 试探点复用"这两个条件的必然结果。黄金比在数学中无处不在（Fibonacci 数列极限、正五边形对角线与边长之比等），此处它以优化算法的形式再次出现。

### 第 ⑥ 页：黄金分割比推导（情况 ★★）

**原文翻译：**

验证情况★★（ρᵏ⁺¹ = λᵏ）也给出同样的方程 β² + β − 1 = 0，解得 β = (−1+√5)/2。

**讲解：**

这一页是对第⑤页的"对称验证"：我们在第⑤页从情况★推导出 β = (−1+√5)/2，现在需要确认情况★★也给出同一个 β 值——否则算法无法同时满足两种情况。

推导过程完全对称：在情况★★中，新区间为 [λᵏ, bᵏ]，匹配条件 ρᵏ⁺¹ = λᵏ 展开后同样化简为 1 − β = β²。这证实了一个 β 值可以同时满足两种情况的匹配条件——黄金分割搜索的自洽性得到了完美验证。

教授将两种情况分开两页推导，既是为了清晰展示，也是为了强调这种对称性不是偶然的。黄金比 β ≈ 0.618 意味着每次迭代区间缩减约 38.2%，或者说保留约 61.8%——比二分搜索的每步减半略差，但每步只需**一次**新函数求值（二分搜索需两次）。综合考虑，黄金分割搜索的"每次函数求值的收敛效率"优于二分搜索。

### 第 ⑦ 页：Golden Section Search 算法与数值示例

**原文翻译：**

数值示例：初始区间 [0, 10]，试探点 3.82 和 6.18；更新后区间 [3.82, 10]，试探点 6.18 和 7.64；再更新后区间 [3.82, 7.64]，试探点 5.28 和 6.18。完整算法四步。注意：别忘了从上一步借用函数值。

**讲解：**

教授先用具体数字展示黄金分割搜索的执行过程，让抽象公式变得具体可触：

- **第1步：** 区间 [0, 10]，宽度 10。λ¹ = 0.618×0 + 0.382×10 = 3.82，ρ¹ = 0.382×0 + 0.618×10 = 6.18。假设 f(λ¹) > f(ρ¹)，新区间 [3.82, 10]，宽度 6.18 ≈ 0.618×10。

- **第2步：** 新的 λ² 应该等于旧的 ρ¹ = 6.18（复用！），只需计算 ρ² = 0.382×3.82 + 0.618×10 = 7.64。

- **第3步：** 假设 f(λ²) ≤ f(ρ²)，新区间 [3.82, 7.64]，宽度 3.82 ≈ 0.618×6.18。新的 ρ³ 等于旧的 λ² = 6.18，只需计算 λ³ = 5.28。

图示清楚地展示了区间的嵌套缩减和试探点的复用。每层宽度都是上一层的 0.618 倍。

最后一句"Don't forget to either borrow f(λᵏ) from f(ρᵏ⁻¹) or f(ρᵏ) from f(λᵏ⁻¹)"是实现时的关键提醒——如果不做这个"借用"，黄金分割搜索就退化为每步两次函数求值，失去了相对于二分搜索的优势。

### 第 ⑧ 页：利用导数的单维优化

**原文翻译：**

这些单维优化算法只使用函数值，不使用导数。如果能获取 f 的导数，可以更高效。配图：在中点 (aᵏ+bᵏ)/2 计算导数。若 f'((aᵏ+bᵏ)/2) ≤ 0（函数递减），新区间取右半 [(aᵏ+bᵏ)/2, bᵏ]。若 f'((aᵏ+bᵏ)/2) > 0（函数递增），新区间取左半 [aᵏ, (aᵏ+bᵏ)/2]。

**讲解：**

教授在最后引入了一个重要的改进思路：如果我们不仅能求函数值，还能求导数，情况会大不同。

前面所有方法（二分搜索、黄金分割搜索）都是"零阶方法"（zero-order methods）——只利用函数值 f(x)，不需要导数信息。但在很多实际问题中（尤其是解析函数），导数是可以计算的。

**基于导数的区间缩减**原理极其简单：在区间中点求导数。如果 f'(mid) ≤ 0，函数在中点仍在递减，说明最小值在中点右边——取右半区间。如果 f'(mid) > 0，函数在中点已在递增，最小值在左边——取左半区间。

这种方法的优势是革命性的：**每步只需一次导数求值**（而非两次函数求值），就能将区间精确减半。这是真正的二分——每步区间缩减 50%，比黄金分割搜索的 38.2% 更好，而且计算量更少。

这也为整个课程建立了一个反复出现的主题：**更多的信息（零阶→一阶→二阶）带来更快的收敛**。后续讲到的 Newton 法将利用二阶信息（Hessian），实现更快的收敛率。
