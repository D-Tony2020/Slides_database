## 讲解大纲

本讲分两大部分：前半收尾无约束优化中的精确线搜索（将一维优化方法应用于梯度下降的步长选择），后半开启约束优化的新篇章——惩罚函数法。

**整体脉络：**

1. **精确线搜索的实际应用**（第①-②页）：将单维优化方法（二分搜索等）嵌入梯度下降，用于自动选择每步的最优步长。
2. **约束优化问题的引入**（第③页）：定义含不等式和等式约束的一般优化问题。
3. **惩罚函数法**（第④页）：核心思想——将约束"搬入"目标函数，通过惩罚违约程度转化为无约束问题。
4. **为什么取平方？**（第⑤页）：解释惩罚项平方的技术原因——保证可微性。
5. **完整算法与示例**（第⑥页）：给出惩罚函数法的迭代算法及具体约束优化示例。

**核心线索：** 从无约束到约束优化的过渡——不是发明全新算法，而是巧妙地将约束问题转化回无约束问题，复用已有的梯度下降工具。

### 第 ① 页：Using Single-Dimensional Optimization to Find Best Step Size During Gradient Descent

**原文翻译：**

使用单维优化来寻找梯度下降中的最佳步长。我们正在用梯度下降最小化 f: ℝⁿ → ℝ。在第 k 次迭代位于 xᵏ，已计算 ∇f(xᵏ)。选择步长 α 使得下一点的函数值尽可能小，即求解 min_{α≥0} f(xᵏ − α∇f(xᵏ))。这是一个单维优化问题。

**讲解：**

这一页将前几讲的内容串联起来：Lecture 3-4 介绍了梯度下降但步长选择留了个"开放问题"，Lecture 4-5 发展了一维优化方法（二分搜索、黄金分割搜索），现在教授把它们合二为一。

精确线搜索的几何意义很直观：梯度下降确定了一个方向（负梯度方向），精确线搜索就是沿着这条"射线"找到使函数值最低的那个点。配图用红色和蓝色清楚地展示了这个过程——从 xᵏ 出发沿 −∇f(xᵏ) 方向走，步长 α 决定走多远。

将 g(α) = f(xᵏ − α∇f(xᵏ)) 视为 α 的一元函数后，Lecture 4-5 介绍的所有一维优化方法都可以直接使用。α 的搜索区间 [0, b] 中 b 需要估计——通常先用试探法找一个足够大的 b 使得 g(b) > g(0)，然后在 [0, b] 上搜索。

### 第 ② 页：用二分搜索选步长的具体示例

**原文翻译：**

在梯度下降第 k 次迭代，位于 xᵏ（已知），已计算 ∇f(xᵏ)（已知）。用单维优化找最佳步长。用二分搜索来寻找最佳步长。示例函数 f(x) = 4ln(e²ˣ¹+e⁴ˣ²) − 6x₁ − 4x₂，使用梯度下降配合二分搜索选步长。

**讲解：**

教授用之前反复出现的 log-sum-exp 函数作为贯穿全课的"running example"，这是很好的教学策略——同一个函数在不同算法中被反复使用，便于比较。

配图将关键概念可视化：将 f(xᵏ − α∇f(xᵏ)) 画成 α 的函数图（红色曲线），然后在 [a, b] 区间上标出二分搜索的试探点 λ 和 ρ。这幅图的教学价值在于让学生看到：虽然原问题是多维的（f 作用于 ℝ² 向量），但步长选择问题确实退化为了一维——α 轴上的一条曲线。

梯度表达式中出现的 8e²ˣ¹/(e²ˣ¹+e⁴ˣ²) 是 softmax 型结构。教授可能会让学生实际用计算机实现这个算法：从某个初始点出发，每步用二分搜索找最佳 α，运行梯度下降，观察收敛过程。

### 第 ③ 页：Constrained Optimization

**原文翻译：**

约束优化。考虑如下形式的约束优化问题：min f(x) s.t. g₁(x) ≤ 0, ..., gₙ(x) ≤ 0, h₁(x) = 0, ..., hₘ(x) = 0，其中所有函数 ℝⁿ → ℝ。在所有我们将考虑的粗糙约束优化方法中，我们将把约束优化问题转化为等价的无约束优化问题，然后使用标准的梯度下降算法。

**讲解：**

这一页标志着课程的重大转折——从无约束优化进入约束优化。教授首先定义了一般的约束优化问题，包含两类约束：不等式约束 gᵢ(x) ≤ 0 和等式约束 hⱼ(x) = 0。

教授的策略非常务实：不急于发展全新的约束优化理论（如 KKT 条件、拉格朗日乘子法等），而是先介绍"粗糙但有效"的方法——将约束问题转化为无约束问题，直接复用梯度下降。这种"转化"思想在优化领域极其重要：很多看似全新的问题，通过巧妙的转化都可以用已有工具解决。

关键词"crude"再次出现——教授一贯用这个词来区分"实用但不完美"的方法和后续将介绍的更精细方法。这也暗示惩罚函数法有局限性，后续会有更好的方法。

### 第 ④ 页：Penalty Function Method

**原文翻译：**

惩罚函数法。将约束搬入目标函数，通过大力惩罚约束的违反。将原约束问题转化为无约束问题：min_x f(x) + Σθᵢ(max{0, gᵢ(x)})² + Σγᵢ(hᵢ(x))²，其中 θ₁,...,θₙ, γ₁,...,γₘ 为大的惩罚参数。

**讲解：**

惩罚函数法的核心思想极其直觉化：如果违反约束"很贵"（大的惩罚参数），那么优化器自然会倾向于满足约束。就像现实生活中，对违规行为设置高额罚款就能减少违规。

具体处理方式：

- **不等式约束** gᵢ(x) ≤ 0：用 θᵢ(max{0, gᵢ(x)})² 惩罚。当 gᵢ(x) ≤ 0（约束满足）时，max{0, gᵢ(x)} = 0，惩罚为零。当 gᵢ(x) > 0（约束违反）时，惩罚为 θᵢ·gᵢ(x)²，违反越多惩罚越大。

- **等式约束** hᵢ(x) = 0：用 γᵢ(hᵢ(x))² 惩罚。只要 hᵢ(x) ≠ 0 就有惩罚，完美满足时惩罚为零。

惩罚参数 θᵢ 和 γᵢ 的选择是实践中的关键问题：太小则约束可能被严重违反，太大则目标函数被淹没，且可能导致数值困难（病态条件数）。教授后面会给出递增惩罚参数的策略。

### 第 ⑤ 页：为什么对不等式约束的惩罚项取平方？

**原文翻译：**

为什么对不等式约束的惩罚项取平方？三幅图展示：g(x) 本身、max{0, g(x)}（不可微）、(max{0, g(x)})²（可微）。在约束边界处 max{0, g(x)} 不可微，但平方后变得可微。

**讲解：**

这是一个技术性但非常重要的细节，教授用三幅图循序渐进地解释。

第一幅图画出原始约束函数 g(x)，在区间 [a, b] 内 g(x) ≤ 0（可行域）。

第二幅图画出 max{0, g(x)}——这是惩罚项的"自然"形式（违反多少罚多少）。问题在于：在 g(x) = 0 的边界点（a 和 b），这个函数有一个"尖角"，即**不可微**。而梯度下降需要计算梯度，不可微意味着算法在边界附近会出问题。

第三幅图画出 (max{0, g(x)})²——平方后尖角消失了！数学原因：在 b 点左侧 r(x) = 0 所以 r'(b⁻) = 0；右侧 r(x) = g(x)² 所以 r'(x) = 2g(x)g'(x)，当 x → b⁺ 时 g(x) → 0，因此 r'(b⁺) = 0。左右导数相等，函数在 b 处可微。

平方不仅保证可微性，还保证了惩罚的"合理性"——小的违反受到小的惩罚（平方放大效应较弱），大的违反受到大的惩罚（平方放大效应强）。

### 第 ⑥ 页：惩罚函数法的完整算法与示例

**原文翻译：**

算法三步：①选初始惩罚参数、容差、放大参数 β>1；②求解带惩罚项的无约束问题得到 xᵏ；③若惩罚项之和 ≤ ε 则停止，否则将所有惩罚参数乘以 β 并回到步骤②。示例：min (x₁−2)⁴ + (x₁−2x₂)² s.t. x₁² − x₂ ≤ 0。

**讲解：**

这一页给出了惩罚函数法的完整迭代框架。关键设计是**递增惩罚参数**：不是从一开始就用极大的 θ 和 γ，而是逐步增大。

为什么不直接用很大的惩罚参数？因为太大的惩罚参数会让目标函数的 Hessian 矩阵条件数变差（ill-conditioning），导致梯度下降收敛极慢或振荡。递增策略的好处是：每次增大惩罚参数后，用上一轮的解作为热启动（warm start），逐步逼近可行解。

停止条件检查的是惩罚项的总和——如果小于 ε，说明约束基本被满足（违反程度微小），可以接受。

示例问题 min (x₁−2)⁴ + (x₁−2x₂)² s.t. x₁²−x₂ ≤ 0 是一个有趣的约束问题：约束要求 x₂ ≥ x₁²（可行域是抛物线上方区域）。转化为无约束问题后，可以用梯度下降求解。教授可能会让学生自行完成这个计算作为练习。

惩罚函数法的局限性在于：解严格来说是不可行的（约束只是"近似"满足）。后续课程中的障碍函数法（barrier method）和增广拉格朗日法会处理这个问题。
