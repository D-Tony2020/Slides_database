## 讲解大纲

本讲围绕两个核心议题展开：梯度下降能保证什么？步长如何选择？

**整体脉络：**

1. **梯度下降算法回顾**（第①页）：回顾朴素梯度下降及试错法选步长。
2. **梯度下降的局限性——局部 vs 全局最小值**（第②-③页）：用具体函数和图示说明梯度下降只能保证收敛到局部最小值。
3. **凸函数的黄金性质**（第④-⑤页）：凸函数中局部最小值即全局最小值；非凸情形的启发式方法。
4. **精确线搜索**（第⑤页下半）：将步长选择转化为一维优化问题。
5. **一维优化方法：区间缩减与二分搜索**（第⑥-⑨页）：介绍不确定区间的迭代缩减策略和二分搜索法。

**核心线索：** 梯度下降的理论保证取决于函数性质（凸 vs 非凸），而步长选择可归结为一维优化子问题。

## 逐页精讲

### 第 ① 页：A Crude Gradient Descent Algorithm for Minimizing a Function

**原文翻译：**

一个用于最小化函数的粗糙梯度下降算法。设 f: ℝⁿ → ℝ 为要最小化的函数。从任意点 x⁰ 出发，沿负梯度方向走小步生成序列。在第 k 次迭代，位于 xᵏ，更新为 xᵏ⁺¹ = xᵏ − αᵏ∇f(xᵏ)。步长大小通过试错法确定：检查函数值是否下降，若否则将步长减半。

**讲解：**

本页是对 Lecture 3 最后一页的完整重述，教授以此作为本讲的出发点。迭代公式 xᵏ⁺¹ = xᵏ − αᵏ∇f(xᵏ) 是梯度下降的"灵魂"——方向由梯度决定，距离由步长决定。

试错法（减半策略）的优点是简单可靠——只要梯度不为零，总能找到足够小的步长使函数值下降。缺点是每步可能需要多次函数求值，效率不高。教授用"crude"一词暗示后续会介绍更精巧的方法——精确线搜索（本讲后半部分）和基于理论的恒定步长（后续课程）。

这一页也设置了本讲的核心问题：这个算法到底能保证什么？能找到全局最优吗？

### 第 ② 页：梯度计算示例与梯度下降的期望

**原文翻译：**

假设要最小化 f(x) = 4·ln(e²ˣ¹ + e⁴ˣ²) − 6x₁ − 4x₂。给出梯度表达式。问：我们对梯度下降能有怎样的最佳期望？配图展示负梯度方向在凸函数两侧的不同情况。

**讲解：**

教授给出了一个具有实际 ML 意义的函数示例——log-sum-exp 型函数减去线性项。这类函数在 softmax 回归中自然出现。梯度表达式涉及 softmax 型的分数（e²ˣ¹/(e²ˣ¹+e⁴ˣ²)），这让学生看到"理论与应用的连接"。

两幅配图展示了一维情况下梯度下降的直觉：无论起点在最小值的左边（导数为负）还是右边（导数为正），负梯度方向总是指向最小值。这就是为什么梯度下降在凸函数上"方向总是对的"——真正的问题只在于步长。

教授接着提出关键问题"What is the best we can expect from gradient descent?"——这为下一页的局部 vs 全局最小值讨论搭建了舞台。

### 第 ③ 页：局部最小值 vs 全局最小值

**原文翻译：**

图示展示非凸函数的局部最小值和全局最小值。全局最小值 x\* 满足 f(x\*) ≤ f(x) 对所有 x 成立。局部最小值 x̂ 满足 f(x̂) ≤ f(x) 对 x̂ 邻域内的所有 x 成立。梯度下降能期望的最好结果是到达一个局部最小值。

**讲解：**

这是理解优化算法局限性的关键一页。教授用一个有多个极值点的非凸函数清楚地展示了问题：梯度下降是一个**局部算法**——它只利用当前位置的梯度信息（一阶局部信息），无法"看到"远处是否有更低的谷底。

教授精确地定义了两个概念：全局最小值是函数在**整个**定义域上的最低点；局部最小值只需在其**邻域**内最低。两者的区别在非凸优化中至关重要。

图中标注的"not a local minimum"很有教学价值——鞍点（梯度为零但非极值）也是梯度下降可能卡住的地方，虽然在一维中鞍点较少出现，但在高维中非常普遍。

教授的结论"Best we can expect from gradient descent is we get to a local minimum"是全讲最重要的定性结论。这解释了为什么深度学习训练存在不确定性——不同的初始化可能导致不同的局部最优。

### 第 ④ 页：凸函数——局部最小值即全局最小值

**原文翻译：**

如果我们最小化的是凸函数，那么全局最小值和局部最小值之间没有区别。所有局部最小值也是全局最小值。配图展示三种情形：唯一最小值的凸函数、碗形二维凸函数、有平坦底部的非严格凸函数。

**讲解：**

这一页是对上一页"坏消息"的"好消息"回应：**如果函数是凸的，梯度下降保证找到全局最优**！

凸函数的定义性质是"任意两点连线段在函数图像之上"（即 f(λx + (1−λ)y) ≤ λf(x) + (1−λ)f(y)）。这保证了不存在"假谷底"——函数曲面像碗一样只有一个（可能不唯一但值相同的）最低处。

教授展示了三种情形来说明凸函数最小值的不同可能：（1）严格凸——唯一最小值点，碗底是一个点；（2）二维凸——碗形曲面，从俯视图看等高线是同心椭圆；（3）非严格凸——底部是一段平坦区域，有无穷多个最小值点，但它们的函数值相同。在所有情形中，任何局部最小值都是全局最小值。

这为后续课程的分析框架定了调：大部分收敛率分析都在凸函数假设下进行。

### 第 ⑤ 页：凸性保证与精确线搜索

**原文翻译：**

如果先验知道目标函数是凸的，就能确信梯度下降算法会成功。当目标函数非凸时，一种启发式方法是用不同初始点多次运行梯度下降。聚焦步长选择问题：当前在 xᵏ，已计算 ∇f(xᵏ)。选择 α 使下一点的函数值尽可能小。即求解 min_{α≥0} f(xᵏ − α∇f(xᵏ))。这是一个关于 α 的单维优化问题。

**讲解：**

这一页承上启下：先总结前面的结论（凸→保证成功，非凸→多起点启发式），然后转入本讲的后半主题——步长选择。

**精确线搜索（Exact Line Search）**的想法极其自然：既然方向（负梯度）已经确定，那就在这条直线上找最优步长——即把 f(xᵏ − α∇f(xᵏ)) 看作 α 的一元函数，最小化它。由于 xᵏ 和 ∇f(xᵏ) 在当前步都是已知的常量，这确实是一个**一维优化问题**。

精确线搜索的优点是理论上每步都是最优的，缺点是需要求解一个子优化问题（可能本身就不容易）。在实践中，通常使用"非精确线搜索"（如 Armijo 条件、Wolfe 条件），它们只需要步长"足够好"而非"最优"。但精确线搜索是理解步长选择的重要起点。

### 第 ⑥ 页：Single Dimensional Optimization

**原文翻译：**

单维优化。有函数 f: ℝ → ℝ，要最小化 f。我们将构造迭代算法。从一个不确定区间 [a¹, b¹]（最小值点在其中）开始，迭代地缩减不确定区间的宽度。

**讲解：**

既然精确线搜索需要求解一维优化问题，教授自然地转入一维优化方法的系统介绍。核心思想是**区间缩减**（interval reduction）：维护一个包含最小值的区间，每次迭代将区间缩小，逐步"逼近"最小值。

示意图非常直观：初始区间 [a¹, b¹] 最宽，每一步生成更窄的嵌套区间 [a², b²] ⊂ [a¹, b¹]，[a³, b³] ⊂ [a², b²]，...直到区间足够窄，可以认为最小值点就在这个小区间内。

这种方法的前提是函数是**单峰**（unimodal）的——在最小值左边递减、右边递增。对于凸函数，这个条件自动满足。

---

### 第 ⑦ 页：迭代缩减不确定区间

**原文翻译：**

如何迭代缩减不确定区间：在第 k 次迭代，当前区间为 [aᵏ, bᵏ]。在区间内取两个试探点——左点 λᵏ 和右点 ρᵏ。计算 f(λᵏ) 和 f(ρᵏ)。

**讲解：**

教授建立了区间缩减算法的通用框架：每次迭代在当前区间内选两个"探测点" λᵏ 和 ρᵏ，通过比较这两点的函数值来决定丢弃区间的哪一部分。选两个点而非一个，是因为一个点的函数值无法告诉我们最小值在哪边——我们需要两个点来建立"比较"。

如何选择 λᵏ 和 ρᵏ 是算法设计的核心——不同的选择策略对应不同的算法（二分搜索、黄金分割搜索、Fibonacci 搜索等）。这为后续内容搭建了框架。

---

### 第 ⑧ 页：区间更新规则

**原文翻译：**

（两幅图）情况①：若 f(λᵏ) ≤ f(ρᵏ)，则最小值在 ρᵏ 左侧，新区间 [aᵏ⁺¹, bᵏ⁺¹] = [aᵏ, ρᵏ]。情况②：若 f(λᵏ) > f(ρᵏ)，则最小值在 λᵏ 右侧，新区间 [aᵏ⁺¹, bᵏ⁺¹] = [λᵏ, bᵏ]。

**讲解：**

这是区间缩减的核心决策规则。逻辑非常清晰，基于单峰函数的性质：

如果 f(λᵏ) ≤ f(ρᵏ)（左点函数值更小），则最小值不可能在 ρᵏ 的右边——因为从 ρᵏ 往右函数值应该继续增大（单峰性）。所以可以安全地丢弃 [ρᵏ, bᵏ] 这段。

反之，如果 f(λᵏ) > f(ρᵏ)（右点函数值更小），则最小值不在 λᵏ 左边，丢弃 [aᵏ, λᵏ]。

无论哪种情况，区间都从 [aᵏ, bᵏ] 缩小了——这保证了算法的收敛。配图用颜色区分新旧区间（蓝色为旧，红色为新），让更新规则一目了然。

### 第 ⑨ 页：Bisection Search for Single Dimensional Minimization

**原文翻译：**

二分搜索用于单维最小化。不同的 λᵏ 和 ρᵏ 选择方式给出不同算法。一种方法：将 λᵏ 和 ρᵏ 对称地放在区间中点两侧，间距为 ε。如果 ε 太小，λᵏ 和 ρᵏ 靠得太近，比较函数值时会遇到数值困难；如果 ε 太大，区间收缩慢。

**讲解：**

二分搜索是最直观的区间缩减策略。将两个探测点放在中点左右各 ε 处：λᵏ = ½(aᵏ+bᵏ) − ε，ρᵏ = ½(aᵏ+bᵏ) + ε。这样每次迭代大约将区间缩减一半（准确地说是缩减到约 ½(bᵏ−aᵏ) + ε）。

教授指出了 ε 选择的两难：ε 太小时虽然每步区间缩减更多，但 f(λᵏ) 和 f(ρᵏ) 的差异可能小到被浮点数精度吞没，导致无法可靠比较——这是数值优化中必须面对的现实问题。ε 太大时每步缩减的比例变差，收敛变慢。

这个 trade-off 也暗示了二分搜索并不是最优的——后续的黄金分割搜索和 Fibonacci 搜索通过更巧妙地选择探测点位置，可以在相同函数求值次数下获得更好的收敛率。
