### 第 ① 页：Deep Learning（回顾）

**原文翻译：**

深度学习。考虑多类分类问题，数据点 {(xⱼ, yⱼ)}，xⱼ ∈ ℝⁿ，yⱼ ∈ {1,2,...,K}。神经网络 L 层结构，每层参数 (Qˡ, gˡ)，激活函数可选 sigmoid 或 ReLU。

**讲解：**

教授再次展示深度网络的结构图作为开场回顾。这是第三次出现这张图（Lecture 2 的第⑦和⑧页也有），反映了这个内容的重要性——教授要确保每个学生都能清楚理解深度网络的前向传播过程。

每一层的操作是：接收上一层输出 aⱼˡ⁻¹ → 线性变换 Qˡaⱼˡ⁻¹ + gˡ → 非线性激活 σˡ(·) → 输出 aⱼˡ。维度标注再次强调了矩阵乘法的兼容性要求。激活函数的两个候选——sigmoid 和 ReLU——对优化的影响截然不同：sigmoid 处处可微但存在梯度消失问题，ReLU 计算简单且梯度不衰减但在原点不可微。这些差异将在后续学习梯度下降时变得更加相关。

### 第 ② 页：深度网络整体表达式

**原文翻译：**

如果使用参数为 (Q¹, g¹, ..., Qᴸ, gᴸ) 的神经网络，对应输入 xⱼ，输出 aⱼᴸ 由函数 z(xⱼ, Q¹, g¹, ..., Qᴸ, gᴸ) 给出，即各层嵌套的复合函数。用 z 作为特征向量求解多类分类问题。

**讲解：**

教授将逐层递推的计算压缩成一个显式的嵌套表达式。花括号标注从内到外逐层展开，让学生直观看到"深度"意味着什么——就是函数嵌套的层数。L 层网络就是 L 次"线性变换 + 非线性激活"的嵌套。

关键的概念跳跃在最后一句：将整个网络输出 z 作为"特征"来做分类。这意味着深度学习可以理解为两步：（1）用神经网络将原始特征 xⱼ 变换为高层特征 zⱼ（特征学习/表示学习）；（2）在 zⱼ 空间上做 softmax 分类（线性分类）。网络的所有参数和 softmax 的参数一起端到端地通过优化来学习——这就是深度学习的"深"：特征不是手工设计的，而是从数据中学到的。

### 第 ③ 页：深度学习的似然函数构建

**原文翻译：**

表格展示每个数据点的深度特征 zⱼ、标签和 softmax 似然。softmax 分类器的参数 (w₁,...,wₖ) 中每个 wₖ ∈ ℝ^(dₗ)。整个数据集的似然是所有数据点似然的乘积，参数包括网络参数和分类器参数。

**讲解：**

这一页将 Lecture 2 中建立的多类逻辑回归框架与深度网络无缝对接。表格的结构与 Lecture 2 第④页完全一致，唯一的区别是"特征向量"列从原始输入 xⱼ 变成了深度特征 z(xⱼ, Q¹, g¹, ..., Qᴸ, gᴸ)。

似然函数的关键变化是：它现在不仅依赖于分类器参数 (w₁,...,wₖ)，还依赖于网络参数 (Q¹, g¹, ..., Qᴸ, gᴸ)。这意味着似然是所有这些参数的函数——优化变量的数量从 K×n（浅层模型）暴增到 K×dₗ + ∑(dˡ×dˡ⁻¹ + dˡ)（深度模型）。对于现代深度网络，这个数量可以达到数百万甚至数十亿。

教授用 zⱼ 作为 z(xⱼ, ...) 的简写，保持表格简洁，但在似然函数的展开形式中又写出了完整的依赖关系——提醒学生不要忘记 z 对网络参数的依赖。

### 第 ④ 页：深度学习的最大似然优化问题

**原文翻译：**

对数似然 = −∑ln(∑eʷₗᵀᶻʲ) + ∑∑𝟙(yⱼ=k)wₖᵀzⱼ。最大似然问题：最大化这个对数似然，优化变量包括每一层的 (Qˡ, gˡ) 和分类器参数 (w₁,...,wₖ)。

**讲解：**

这是前三讲"建模"阶段的终点——深度学习的完整优化问题终于写出来了。教授特别在左侧逐行列出了所有优化变量的维度，让学生清楚看到参数空间的结构。

这个优化问题的核心特征是**非凸性**。虽然对数似然对 softmax 参数 (w₁,...,wₖ) 是凹的（当特征固定时），但由于 zⱼ 本身是网络参数的高度非线性函数（多层嵌套），整体目标函数关于全部参数是非凸的。这意味着：（1）可能存在多个局部最优；（2）不同的初始化可能导致不同的解；（3）无法保证找到全局最优。

然而实践中，梯度下降（及其变体）在训练深度网络时表现出人意料地好——理解为什么，是现代优化理论的活跃研究方向。这门课在后续会逐步建立分析凸和非凸优化的工具。

### 第 ⑤ 页：Basics of Gradient Descent Methods

**原文翻译：**

梯度下降方法基础。设 f: ℝⁿ → ℝ 是将 n 维向量映射到标量的函数。f 在点 x⁰ 处的梯度是 n 维向量 ∇f(x⁰) = (∂f/∂x₁|ₓ₌ₓ₀, ..., ∂f/∂xₙ|ₓ₌ₓ₀)。一般原理：从 x⁰ 沿梯度方向走小步，函数值增大；逆梯度方向走小步，函数值减小。

**讲解：**

经过三讲的"问题定义"，终于进入"如何求解"的部分。教授从最基本的概念——梯度——讲起。

梯度 ∇f(x⁰) 是一个向量，每个分量是函数对对应变量的偏导数，在 x⁰ 处求值。几何意义：梯度指向函数**最快增长**的方向。因此要最小化函数，应该沿**负梯度方向**（−∇f(x⁰)）移动。

教授的配图非常直观：在一维情况下，x⁰ 处的梯度（斜率）指向函数上升的方向（向左），负梯度指向下降的方向（向右）。步长 α 控制每一步走多远。这个简单的一维图蕴含了梯度下降的全部直觉。

关键强调"for a small step size"——步长必须足够小。这个限定词至关重要，因为梯度只提供**局部**的方向信息，走太远就可能越过最小值甚至使函数值增大。下一页的数值例子将生动说明这一点。

---

### 第 ⑥ 页：梯度下降的数值示例

**原文翻译：**

如果 α 足够小，则 f(x⁰ − α∇f(x⁰)) ≤ f(x⁰)。例：f(x) = x₁² + 4x₁x₂ + 3x₂² − 5，梯度为 (2x₁+4x₂, 4x₁+6x₂)ᵀ。从 x⁰=(1,1)ᵀ 出发，步长 0.1，到达 (0.4, 0)ᵀ。f 从 3 降到 −4.84，确认函数值下降。

**讲解：**

教授选择了一个二次函数作为示例——足够简单以便手算，但包含交叉项 4x₁x₂ 使得等值线不是简单的圆（而是椭圆），增加了一点复杂性。

计算过程清晰明了：首先求梯度 ∇f 的表达式（对每个变量求偏导），然后在 x⁰=(1,1)ᵀ 处代入得 (6, 10)ᵀ，接着执行更新 x¹ = (1,1)ᵀ − 0.1×(6,10)ᵀ = (0.4, 0)ᵀ。

关键验证：f(x⁰) = 1 + 4 + 3 − 5 = 3，f(x¹) = 0.16 + 0 + 0 − 5 = −4.84。确实 3 > −4.84，函数值下降了。步长 α = 0.1 足够小，梯度下降有效。

这个例子的教学意图是：让学生亲手"体验"一步梯度下降的完整流程——求梯度、代入、更新、验证。每个学生都应该能复现这个计算。

### 第 ⑦ 页：步长过大的反例

**原文翻译：**

同一起点 x⁰=(1,1)ᵀ，步长改为 0.5。新点为 (−2, −4)ᵀ。f 从 3 变成 79，函数值不降反升！

**讲解：**

这是与上一页的完美对照实验——唯一的变化是步长从 0.1 增大到 0.5，结果截然相反。新点 (−2, −4)ᵀ 的函数值 79 远大于起点的 3，梯度下降完全失败。

为什么会这样？因为步长 0.5 太大了。梯度只告诉我们在 x⁰ 附近的**局部**下降方向，但走了 0.5×(6,10) = (3, 5) 这么远的距离后，我们已经远远超出了梯度的"有效范围"。函数在远处的行为可能完全不同——在这个例子中，二次函数在远处急剧增长。

这个反例传递了一个核心教训：**梯度下降的成败取决于步长的选择**。步长太大会导致发散（函数值爆炸），步长太小虽然安全但收敛极慢。如何选择"恰到好处"的步长？这是优化理论的核心问题之一，后续课程会系统讨论。

### 第 ⑧ 页：步长选择的图示

**原文翻译：**

（图示）一维凸函数曲线上展示两种步长的效果：小步长时新点落在函数值更低处（成功）；大步长时新点越过最小值，函数值反而增大（失败）。当 ∇f(x⁰) ≤ 0 时，x⁰ − α∇f(x⁰) ≥ x⁰。

**讲解：**

教授用一维曲线图将第⑥⑦页的数值发现可视化。这张图是理解步长选择的"黄金图示"：

在 x⁰ 处函数递减（∇f(x⁰) < 0），负梯度方向是向右。小步长时，新点停在曲线的下降段，函数值确实减小——这是梯度下降正常工作的情况。但步长过大时，新点"飞"过了最小值，落在了曲线另一侧的上升段，函数值反而比起点还高。

底部的分析给出了一维情况的直觉：当导数为负时，−α·∇f(x⁰) 为正，所以 x 增大（向右移动），这是正确的方向。但走多远才合适？图上清楚地显示，存在一个"安全范围"，超过就会过冲。

这张图虽然是一维的，但多维情况下的原理完全一样——只是"过冲"可能发生在任意方向，使得问题更加复杂。

### 第 ⑨ 页：梯度下降算法

**原文翻译：**

我们立即得到一个粗糙的最小化算法。从任意点 x⁰ 出发，通过沿负梯度方向走小步，生成点序列 x¹, x², x⁵, ...。在第 k 次迭代，位于点 xᵏ，使用步长 αᵏ。更新公式：xᵏ⁺¹ = xᵏ − αᵏ∇f(xᵏ)。通过"试错法"找 αᵏ：选任意值，检查函数值是否下降；若是，接受该步；若否，将 αᵏ 减半重试。

**讲解：**

这一页给出了梯度下降算法的第一个完整描述。教授诚实地称之为"crude algorithm"（粗糙的算法），因为步长选择策略是最朴素的**回溯线搜索**（backtracking line search）：尝试一个步长，如果函数值下降就接受，否则减半重试。

算法的三个核心组件：（1）**迭代公式** xᵏ⁺¹ = xᵏ − αᵏ∇f(xᵏ)——每步沿当前位置的负梯度方向移动；（2）**步长选择**——通过检验 f(xᵏ⁺¹) < f(xᵏ) 来确保每步都有"进步"；（3）**终止条件**（此处未明确说明，后续课程会讨论）。

"减半"策略保证了搜索一定会终止：不断减半步长，最终一定能找到足够小的步长使得函数值下降（前提是梯度不为零）。但这个策略效率不高——每次迭代可能需要多次函数求值来确定步长。后续课程将介绍更高效的步长选择策略，包括恒定步长（基于 Lipschitz 常数）和精确线搜索。

教授用这个"粗糙但可用"的算法作为起点，后续逐步精化——这体现了工程教育中"先能用，再优化"的实用主义教学理念。
