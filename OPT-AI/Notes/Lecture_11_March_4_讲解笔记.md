# Lecture 11 — March 4, 2026 — 讲解笔记

## 讲解大纲

本讲的核心主题是**梯度下降法的收敛性分析**，全部围绕一个中心问题展开：梯度下降到底能多快地逼近最优解？

**整体脉络：**

1. **回顾起点**（第①页）：回顾此前已推导的梯度下降累积误差预备不等式，作为后续所有分析的基石。
2. **一般凸函数的收敛率**（第②-③页）：在仅假设凸性和梯度有界的条件下，证明 O(1/√K) 的收敛率，并推导迭代复杂度——精度提高 10 倍需要迭代次数增加 100 倍，代价昂贵。
3. **引入光滑性**（第④页）：定义 L-光滑凸函数——函数值被线性近似（下界）和二次近似（上界）夹住。这一更强的假设将带来显著更好的收敛率。
4. **两个预备结论**（第⑤-⑥页）：Result 1 利用光滑性证明每步迭代目标函数值必然下降（单调性）；Result 2 重申此前的累积误差界。
5. **光滑凸函数的收敛定理**（第⑦-⑧页）：将两个 Result 巧妙结合，证明 O(1/K) 收敛率——比一般凸情形快得多。
6. **迭代复杂度对比与渐近收敛**（第⑨页）：光滑情形下精度提高 10 倍仅需迭代次数增加 10 倍（线性关系 vs 平方关系），并证明梯度下降可渐近收敛到最优。

**关键对比线索：** 一般凸 O(1/√K) vs 光滑凸 O(1/K)，这一对比贯穿整讲，是理解"为什么光滑性如此重要"的核心。

## 逐页精讲

---

### 第 ① 页：Convergence Results for Gradient Descent

**原文翻译：**

设 f: ℝⁿ → ℝ 为凸函数，具有全局最小值点 x\*。我们用恒定步长 γ 的梯度下降来最小化 f。迭代公式为 xᵏ⁺¹ = xᵏ − γ·gᵏ，其中 gᵏ = ∇f(xᵏ)。预备分析：我们之前做了一个预备分析，以界定梯度下降在 K 次迭代中的累积"误差"。

**讲解：**

这一页是整讲的"地基"。教授首先清晰地摆出了问题设定：我们有一个凸函数 f，它在某个点 x\* 处取得全局最小值，我们要用梯度下降去找这个最小值点。梯度下降的规则非常简单——每一步沿着当前位置的负梯度方向走一个固定大小的步长 γ。

然后教授回顾了之前课程中已经推导出的一个关键预备不等式。这个不等式的左边是所有迭代点的函数值与最优值之差的累积和——可以理解为"K 次迭代中犯的总误差"。右边由两项组成：第一项 (γ/2)∑‖gᵏ‖² 与梯度的大小有关，反映了每一步走的"力度"；第二项 (1/(2γ))‖x⁰−x\*‖² 是初始点离最优解的距离，除以步长。这个不等式的直觉是：总误差一方面被梯度步的大小推高（走太猛可能"过冲"），另一方面被起点离目标的距离推高（出发点越远误差越大），而步长 γ 在两者之间起到平衡作用——γ 大了第一项大但第二项小，γ 小了反之。这为后续选择最优步长埋下伏笔。

### 第 ② 页：一般凸函数的收敛定理

**原文翻译：**

定理：设 f: ℝⁿ → ℝ 凸，具有全局最小值点 x\*，且 ‖∇f(x)‖ ≤ B。假设 ‖x⁰−x\*‖ ≤ R，且我们使用恒定步长 γ = R/(B√K) 的梯度下降。则梯度下降的迭代满足：(1/K)∑(f(xᵏ)−f(x\*)) ≤ RB/√K。由该定理可得，K 次迭代中的最佳解的函数值与最优值之差不超过 RB/√K。如果我们运行 K 次梯度下降，沿途遇到的最佳解的目标函数值与最优目标值之差至多为 RB/√K。

**讲解：**

这是本讲第一个正式的收敛定理。它需要两个假设：（1）梯度有界——∇f 的范数处处不超过某个常数 B，这限制了函数的"陡峭程度"；（2）初始点离最优解的距离不超过 R。

关键技术是**步长的选择**：γ = R/(B√K)。注意这个步长依赖于总迭代次数 K——你需要"提前规划"要跑多少步，才能设定最优的步长。这是一般凸情形下的一个局限性：步长选择需要先验知识。

定理的结论是**平均误差**以 O(1/√K) 的速率收敛到零。注意这里说的不是最后一个迭代点的误差，而是所有迭代点的平均误差。但因为"最佳 ≤ 平均"这一简单事实，K 步中最好的那个点的误差同样被 RB/√K 所控制。

O(1/√K) 的收敛率意味着：要把误差缩小一个数量级（比如从 1 缩到 0.1），迭代次数需要增加两个数量级（100倍）。这个速率相当慢，但它是在"仅仅假设凸性"这一非常弱的条件下得到的——后面我们会看到，加上光滑性假设后收敛会快得多。

---

### 第 ③ 页：迭代复杂度（一般凸函数情形）

**原文翻译：**

反过来问一个类似的问题：假设我们希望获得误差不超过 ε 的解，需要运行多少次梯度下降？令 ε = RB/√K，解得 K = (RB/ε)²。如果 ε = 0.1，K = 常数·100；如果 ε = 0.001，K = 常数·1000000。迭代次数随精度要求的提高而快速增长。

**讲解：**

这一页将上一页的收敛率定理"反过来看"——给定精度要求 ε，需要多少步？这就是**迭代复杂度**（iteration complexity）的概念。

从 ε = RB/√K 反解出 K = (RB/ε)²，立刻看到 K 与 1/ε² 成正比。教授用两个具体的数字来让学生直观感受：要达到 ε = 0.1 的精度需要大约 100 步迭代，但要达到 ε = 0.001——精度提高了 100 倍——需要 1000000 步，即迭代次数增加了 10000 倍！这是因为 K 与 ε 之间是**平方关系**：精度每提高 10 倍，迭代次数增加 100 倍。

教授的结论是"迭代次数随精度要求的提高而快速增长"，这暗示了一般凸函数仅用梯度下降来求解在高精度需求下代价极其昂贵。这自然地引出了下一个问题：能否在更强的假设下获得更好的收敛率？答案是肯定的——光滑凸函数。

### 第 ④ 页：Convergence of Gradient Descent for Smooth Convex Functions

**原文翻译：**

梯度下降对光滑凸函数的收敛性。定义：设 f: ℝⁿ → ℝ 为凸函数，我们说 f 是参数为 L 的光滑函数，如果 f(y) ≤ f(x) + ∇f(x)ᵀ(y−x) + (L/2)‖y−x‖² 对所有 x, y ∈ ℝⁿ 成立。另外回忆对凸函数 f，有 f(y) ≥ f(x) + ∇f(x)ᵀ·(y−x)。直觉上，光滑凸函数位于局部二次近似之下、局部线性近似之上。

**讲解：**

这是本讲的转折点——从"一般凸"进入"光滑凸"。光滑性（L-smoothness）的定义初看是一个不等式，但其含义极为深刻：它说的是函数在任意一点 x 处，被一个以该点为中心的**二次抛物线**从上方"罩住"。参数 L 控制这个抛物线的"开口大小"——L 越大，允许的曲率越大，函数可以变化得越剧烈。

配合凸性给出的下界 f(y) ≥ f(x) + ∇f(x)ᵀ(y−x)，我们得到一个漂亮的"三明治"结构：**线性近似 ≤ f(y) ≤ 二次近似**。函数被切线从下方"托住"，被二次曲线从上方"盖住"。教授的配图非常直观地展示了这一点：实际函数曲线夹在两条辅助曲线之间，三者在 x 处相切。

为什么这个性质这么有用？因为它让我们能精确控制每一步梯度下降的行为——我们可以保证每一步目标函数值一定会下降，而且下降的量可以精确量化。这在一般凸函数中是做不到的（一般凸函数的梯度可能非常大，导致步子迈过头反而上升）。

等价理解：L-光滑也可以表述为梯度的 Lipschitz 连续性：‖∇f(x) − ∇f(y)‖ ≤ L‖x−y‖，即梯度不会"突变"。

### 第 ⑤ 页：预备结论 1 —— 单步下降引理

**原文翻译：**

我们需要两个预备结果来分析光滑凸函数的梯度下降。Result 1：设 f: ℝⁿ → ℝ 光滑且凸，x⁰, x¹, x², ... 由恒定步长 γ 的梯度下降生成。则 f(xᵏ⁺¹) ≤ f(xᵏ) − γ(1−γL/2)‖gᵏ‖²。证明：由光滑性，展开并代入梯度下降更新公式即得。特别地，当 γ = 1/L 时，f(xᵏ⁺¹) ≤ f(xᵏ) − (1/(2L))‖gᵏ‖²。

**讲解：**

Result 1 是整个光滑凸分析的核心引理。它告诉我们一个极其重要的事实：**只要步长选得合适，梯度下降的每一步目标函数值都一定会下降**，而且下降的量正比于当前梯度的范数的平方。

证明过程是教科书级别的"代入消去"：先用光滑性定义把 f(xᵏ⁺¹) 用 f(xᵏ) 加上一阶项和二阶项来上界，然后把梯度下降的更新公式 xᵏ⁺¹ − xᵏ = −γgᵏ 代入。红色标注清楚地标出了每个替换：∇f(xᵏ) 就是 gᵏ，位移量就是 −γgᵏ，位移的范数就是 γ‖gᵏ‖。代入后整理，就得到了这个漂亮的单步下降不等式。

关键观察：括号中的 (1 − γL/2) 必须为正才能保证下降，这要求 γ < 2/L。当取 γ = 1/L 时，括号值为 1/2，得到最简洁的形式：每步至少下降 (1/(2L))‖gᵏ‖²。这个结果有两个重要推论：（1）迭代序列的函数值**单调递减**；（2）如果某一步梯度很大（离最优解远），则该步下降幅度也大，这是一种"自适应"的性质。

### 第 ⑥ 页：预备结论 2

**原文翻译：**

Result 2：这就是我们此前的预备分析：∑(f(xᵏ)−f(x\*)) ≤ (γ/2)∑‖gᵏ‖² + (1/(2γ))‖x⁰−x\*‖²。

**讲解：**

这一页非常简短，只是重新声明了第 ① 页已经回顾的那个预备不等式。教授特意单独列一页来强调它，是因为接下来的定理证明需要**同时**用到 Result 1 和 Result 2，将两者巧妙地结合在一起。

回忆一下，这个不等式的含义是：K 步梯度下降的累积误差被两项控制——梯度项和初始距离项。在之前的一般凸分析中，我们只能用梯度有界（‖gᵏ‖ ≤ B）来粗暴地上界梯度项。但现在有了 Result 1 给出的精细控制（每步梯度范数的平方可以被函数值的下降量所界定），我们可以做得好得多。这就是接下来两页定理证明的关键思路。

### 第 ⑦ 页：光滑凸函数的梯度下降收敛定理

**原文翻译：**

定理：设 f: ℝⁿ → ℝ 为参数为 L 的光滑凸函数，具有全局最小值点 x\*。假设 ‖x⁰−x\*‖ ≤ R。选择步长 γ = 1/L，梯度下降算法的迭代满足：f(xᴷ)−f(x\*) ≤ (L/(2K))·R²。证明开始：由 Result 1，对不等式在 k=0,...,K−1 上求和并利用伸缩求和，得到梯度范数平方和的上界。再代入 Result 2。

**讲解：**

这是本讲最重要的定理。与第 ② 页的一般凸定理相比，它有三个关键改进：

**第一**，收敛率从 O(1/√K) 提升到了 **O(1/K)**。这是质的飞跃——精度每提高 10 倍，迭代次数只需增加 10 倍而非 100 倍。

**第二**，结论针对的是**最后一个迭代点** xᴷ，而非"最佳迭代点"。这在实践中更有用——你不需要保存所有中间结果来找最好的那个。

**第三**，步长 γ = 1/L **不依赖于迭代次数 K**。你不需要提前知道要跑多少步——只要知道光滑参数 L 就够了。这在实际应用中方便得多。

证明的关键技巧是"两路夹击"：Result 1 给出 ∑‖gᵏ‖² 的上界（通过伸缩求和消去中间项，得到 ≤ 2L(f(x⁰)−f(xᴷ))），Result 2 给出累积误差 ∑(f(xᵏ)−f(x\*)) 的上界（用 ∑‖gᵏ‖² 表达）。将前者代入后者，就能消去 ∑‖gᵏ‖² 这个中间量。

---

### 第 ⑧ 页：定理证明（续）—— 从平均误差到末端误差

**原文翻译：**

（接上页推导）化简得 ∑ₖ₌₁ᴷ(f(xᵏ)−f(x\*)) ≤ (L/2)R²，进而 (1/K)∑ ≤ (L/(2K))R²。由 Result 1 知函数值序列单调递减：f(x¹) ≥ f(x²) ≥ ... ≥ f(xᴷ)，因此 f(xᴷ)−f(x\*) 是最小的一项，不超过平均值。得证。

**讲解：**

这一页完成了定理的证明，用到了一个非常巧妙的技巧。

首先，经过代数化简（将第⑦页最后一行的不等式展开，f(x⁰)−f(x\*) 和 −f(x⁰) 相消，f(xᴷ) 和 −f(x\*) 合并），得到一个干净的结论：从 k=1 到 K 的累积误差总和不超过 (L/2)R²。除以 K 就得到了平均误差的上界。

但我们要的是 f(xᴷ)−f(x\*) 的界，不是平均值的界。这里用到了 Result 1 的一个推论——函数值序列**单调递减**。既然 f(xᴷ) 是序列中最小的，那么 f(xᴷ)−f(x\*) 就是所有 f(xᵏ)−f(x\*) 中最小的一项。而一组非负数中最小值不超过它们的平均值——这个简单的观察完成了最后一步。

教授在末尾写下 "Done!" 来标记证明完成。整个证明链条清晰：光滑性 → 单步下降 → 伸缩求和 → 代入预备不等式 → 化简 → 单调性得末端界。

---

### 第 ⑨ 页：迭代复杂度（光滑凸函数情形）与渐近收敛

**原文翻译：**

需要运行多少次梯度下降才能使最后一步迭代的函数值与最优值之差不超过 ε？令 ε = (L/(2K))R²，解得 K = LR²/(2ε)。若 ε = 0.1，K = 常数·10；若 ε = 0.001，K = 常数·1000。此外，0 ≤ f(xᴷ)−f(x\*) ≤ (L/(2K))R²，令 K→∞，由夹逼定理，f(xᴷ)→f(x\*)。因此若运行梯度下降足够长，最后的迭代将任意接近最优。

**讲解：**

这是全讲的高潮和总结。教授将光滑凸情形的迭代复杂度与第③页的一般凸情形做了隐含的对比，效果令人印象深刻：

| 精度 ε | 一般凸 K = O(1/ε²) | 光滑凸 K = O(1/ε) |
|--------|---------------------|---------------------|
| 0.1    | ~100                | ~10                 |
| 0.001  | ~1,000,000          | ~1,000              |

光滑情形下 K 与 1/ε 是**线性关系**：精度提高 10 倍，迭代次数也只增加 10 倍。对比一般凸的平方关系，效率提升是天壤之别。在实践中，大多数有意义的优化问题（如神经网络训练中的损失函数）都至少近似满足光滑性，这就是为什么梯度下降在实践中效果远好于最坏情况理论预测。

最后教授用**夹逼定理**（Squeeze Theorem）给出了渐近收敛的证明：f(xᴷ)−f(x\*) 夹在 0 和 (L/(2K))R² 之间，后者趋于零，所以函数值序列收敛到最优值。这个结论虽然看起来"理所当然"，但它的严格性很重要——它保证了梯度下降（选择 γ=1/L）对光滑凸函数不仅是近似收敛，而是**精确收敛**到全局最优。

总结全讲：光滑性这一额外假设带来了从 O(1/√K) 到 O(1/K) 的收敛率提升，从需要提前知道 K 到步长独立于 K 的实用性提升，以及从"最佳迭代"到"最后迭代"的结论增强。这三重改善构成了现代优化理论的重要基石。
